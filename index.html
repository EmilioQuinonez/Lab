Preguntas

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es son todas las computadoras que estan conectadas entre si y la web son paginas que estan sobre esa red de computadoras.

2. ¿Cuáles son las partes de un URL?

Son protocolo, subdominio, dominio, ruta, parametros para la ruta y etiqueta.

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

El Get es para obtener o recuperar la informacion, el Head es similar al GET pero sin el cuerpo de la respuesta, POST se usa para enviar informacion al servidor y generar un cambio, el PUT remplaza la informacion seleccionada por una diferente, el PATCH se utiliza para aplicar modificaciones parciales y por ultimo DELETE borra un recurso en especifico.

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El metodo que se utiliza es el POST ya que estas enviando informacion al servidor.

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El metodo que se utiliza es el GET porque este nos sirve para consultar la informacion del servidor.

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Cuando devuelve un codigo 200 significa que la accion se realizo de manera correcta sin ningun error.

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No seria su responsabilidad, ya que el error 404 es debido a problemas de conexion al servidor, es decir que al momento que tu trataste de ingresar al servidor trataste de entrar a un apartado que no existe en el.

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si es responsabilidad del desarrollador, ya que este error implica que la solicitud que se intento hacer fue valida pero fallo el servidor en realizar el la accion.

9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Que este depreciado o desaprobado es que no se recomienda su uso, algunos elementos de html 4 que ya no funcionen en html 5 es con las etiquetas dir y bgsound, que fueron cambiadas por ul y audio respectivamente.

10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Una de las mayores diferencias es que html 5 es compatible con navegadores mas nuevos y no es compatible con la mayoria de navegadores viejos, ademas que algunas etiquetas cambiaron o no estan dsiponibles en html4.

11. ¿Qué componentes de estructura y estilo tiene una tabla?

Los componentes que tiene una tabla son cuerpo, encabezado y pie.

12. ¿Cuáles son los principales controles de una forma HTML5?

Los principales controles de una forma html5 son los botones, campos de entrada, cuadros de seleccion area de texto, etc.

13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

Ya que como Opera esta basado en el motor Chromium, Opera admite un conjunto completo de funciones HTML5.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

14. ¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sitemas de informacion estan establecidos por 7 pasos, los cuales son, planificacion, analisis, diseño, implementacion, pruebas, instalacion o despliegue y uso o mantenimiento.

15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo consiste en diseño, codificacion, testeo y liberación.
